# no-bang

## 概述

`no-bang` 是一个自定义 ESLint 规则，用于强制禁止在 JavaScript 或 TypeScript 文件中使用逻辑非运算符（`!`）。此规则帮助开发人员避免因不当使用 `!` 造成的逻辑错误，特别是在处理数字、布尔值或空值时，确保代码更加健壮和可预测。

## 功能

- **禁止使用 `!` 运算符**: 自动检查代码中所有 `!` 运算符的使用，特别是在条件判断中，避免出现因为 `0` 或 `null` 产生误判的情况。
- **报告 `!` 运算符的使用**: 规则会报告所有使用 `!` 运算符的地方，并建议开发者采用更明确的比较方式。
- **增强代码的可读性和安全性**: 禁止使用 `!` 可以减少潜在的错误，特别是在处理空值、数字、布尔值时，减少开发人员的理解负担。

## 示例

### 有效示例

```typescript
// 使用明确的比较方式，避免使用 `!`
if (x === null) {
  // 执行当 x 是 null 时的逻辑
}
```

### 无效示例

```typescript
// 使用 `!` 运算符进行判断
if (!x) {
  // 当 x 是 null 或 0 时，都会进入此条件，可能不是预期的行为
}
```

在无效示例中，ESLint 会报告错误，因为使用了 `!` 运算符，并建议将其替换为更明确的条件判断。

### 自动修复

目前该规则不支持自动修复。你需要手动修改代码，使用明确的比较方式来替代 `!` 运算符。

## 配置

此规则没有任何选项。启用后，插件会强制在所有 JavaScript 或 TypeScript 文件中禁止使用 `!` 运算符。

启用该规则后，任何使用 `!` 运算符的代码都将会被标记为错误，并需要进行修改。
